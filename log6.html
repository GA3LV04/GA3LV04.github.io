Code References

For this assignment, I looked up CSS Flexbox. Flexbox is a layout system in CSS that helps organize content into rows or columns and makes it easier to design responsive web pages.

Websites I Used

Here are the four resources I used to research Flexbox:

  • MDN Web Docs – Flexbox
  • W3Schools – CSS Flexbox
  • CSS-Tricks – Flexbox Guide
  • web.dev – Flexbox

Differences Between the Sources

Each website explained Flexbox differently. MDN Web Docs had the most detailed and technical explanation and it explained how Flexbox works as a one-dimensional layout system and included a lot of definitions and terminology.

W3Schools was much simpler and easier to understand and it focused on short explanations and basic examples, which made it good for beginners but not as detailed.

CSS-Tricks was very visual and organized and it showed both container and item properties in one place and made it easier to understand how everything works together.

web.dev focused more on teaching and real-world use and it explained how Flexbox is used in modern web design and how it helps create responsive layouts.

What Makes a Good Code Reference Page?

I think a good code reference page should include clear explanations, simple examples, and visuals that show how the code works. It should also be easy to navigate so users can quickly find what they need.

Having both beginner-friendly explanations and more detailed information is important so different levels of users can understand the content.

Best and Worst Resource

Out of the four, CSS-Tricks stood out the most because it was very visual and easy to understand. MDN was also very helpful but more technical.

W3Schools was the easiest to read, but it did not go as deep into the topic. web.dev was useful because it connected Flexbox to real-world design.
